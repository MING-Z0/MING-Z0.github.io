关键字

  • 被语言赋予特定含义的单词，用来对其他内容的修饰

  • 关键字中所有字母都为小写保留字

    保留字

    • 现版本java尚未使用，但是以后版本可能作为关键字使用
    • goto const 等

标识符

  • 定义规则
    • 由个26字母的大小写，0-9，_或$组成，数字不能作为开头
    • 严格区分大小写，长度无限制
    • 不能包含空格
  • 约定俗成：
    • 类名首字母大写
    • 包名、变量名、方法名都小写
    • 见名知意
    • 变量名、方法名：多个单词组成时，首字母小写，其他单词首字母大写（驼峰）
    • 常量名：所有字母大写 XXX_YYY_ZZZ

变量

  • 变量表示内存中的一个存储区域，类型不同，占用的空间大小不同。
  • 变量在同一个作用域内不能重名。

数据类型

  • 指导数据的存储
  • 分类

整数类型

==java中整数字面量为int类型==

浮点类型

==浮点型字面量为double==

需要小心浮点数的小数大小判断，精度丢失

==java基本数据类型不受具体操作系统的影响==

逻辑型（布尔值boolean）1个字节

==不能用0或1表示，java中只能用true和false表示==

字符型

占2个字节 表示一个字符

只能使用单引号 ‘ ‘

==基本类型参与运算时会转为容量更大的类型==

八种基本数据类型的转换

  • java中可以从任意基本类型转型到另外的基本类型

  • ==例外 : boolean 类型不可以转换为其他的数据类型。==

  • 转换分为==默认转换==和==强制转换==

  • 容量大的数据类型转换为容量小的数据类型时，要加上强制转换符，但可能造成精 度降低或溢出

  • 有多种类型的数据混合运算时，系统首先自动的将所有数据转换成容量最大的 那一 种数据类型，然后再进行计算

==byte、short、char在运算时会默认转换为int类型==

原因：JVM设计问题，优化指令集

类

类的结构：

  • 成员变量
  • 方法
  • 构造方法：初始化对象
  • 内部类：在类体中声明的类
  • 块：一段没有名称的代码块

对象

对象是根据类创建出来的实例

构造方法

用来初始化对象的方法

特征：

  • 每个类中默认都有一个无参的构造方法
  • 构造方法名必须与类名相同
  • 构造方法无返回值，且不需要void修饰

作用：

  • 创建类时，对对象进行初始化
  • 成员变量的默认值，就是在无参构造方法里赋值的

==注意==：

方法重载

参数个数、参数顺序、参数类型来区分重载类型

返回值对方法重载没影响

普通成员方法也可以重载

对象与引用

变量存储的只是对象在内存（堆区）中的地址，所以这种变量被称为引用数据类型

值传递问题

基本数据类型是值传递，引用类型是地址传递

关键字

this：

同c++中，用来表明是这个对象的成员

static：

修饰成员属性

  • 静态

    ​ 静态成员对象在内存中只有一份，和类信息在方法区中存储

  • 随着类的加载而加载：

    ​ 只要使用到了类，类就会被加载

  • 优先于对象存在（未创建对象也可使用）

  • 修饰的成员被所有对象共享

  • 可以不创建对象，直接使用类调用

修饰成员方法：
  • 不能使用非静态的成员变量（其他变量不能随类的加载而加载）
  • 不能使用 ==this== 关键字
  • 静态方法存储在？

代码块

  • 类似一个没有名称的方法

  • 可以用修饰符static修饰

  • 代码块在满足条件是，自动调用执行

    • 非静态：

    类加载时被调用

    • 静态：

    在创建类对象时被调用，只执行一次

包

为了更好的组织类，java设计了包机制，区别类名的命名空间

命名规范:

导入外部包的类，关键字 import

控制访问权限

  • public：公共权限 修饰类、属性、方法。可以被任意访问
  • protected：受保护的权限 修饰属性、方法。可以被同包类访问，如果不是同包类，必须是该类的子类才可以访问。
  • default：同包权限 修饰类、属性、方法。只能被同包的类访问
  • private：私有权限 修饰属性、方法 只能在本类中访问。

封装

将类的某些信息==隐藏==在类内部，不允许外部程序直接访问，而是通过该类提供的方法来时间对隐藏信息的操作和访问。

  • 隐藏类的实现细节
  • 只能通过规定方法访问
  • 方便加入控制语句
  • 方便修改实现

set 和 get 方法

继承

子类可以拥有父类部分功能

子类可以在父类功能上进行扩展

提高代码的复用性

java中如果一个类没有继承其他类，则Jvm会让这个类==默认继承Object类==

基本语法
继承中的构造方法

子类创建时，会先调用父类的构造方法

super关键字
super关键字作用：
  • 使用super关键字来访问父类成员
    • super.成员变量引用父类成员变量
    • super.方法名(参数列表)访问父类方法
    • super.构造方法(参数列表)访问父类构造方法

==不要把super误认为是父类对象，在创建子类对象时，不会创建父类对象。只会将父类信息加载到子类对象中存储==

方法重写

方法的覆盖

当父类方法的具体实现不能满足子类需求是，可以在子类中定义与父类中结构相同的方法

重写规则
  • ==访问权限不能小于父类权限==
  • 方法名相同，返回值、参数列表完全一致

注意：构造方法、静态方法不能重写，成员变量不存在重写。

抽象类

一个只有方法结构，没有方法体的方法

（只对方法进行声明，没有实现）

abstract关键词修饰

==同时需要将类使用abstract修饰成抽象类==

抽象类不能用来创建对象

继承的非抽象子类不许实现全部的抽象方法

多态

必要条件

  • 要有继承（包括接口的实现）(前提条件)
  • 要有重写）(前提条件)
  • 父类引用指向子类对象

对非静态的方法：编译看左边，运行看右边

对静态的方法：编译和运行都看左边

用一个父类的引用变量可以表示任意子类对象，实现代码的可扩展性

多态转型

父类表示任意的子类：向上转型

  • 提高程序的扩展性

重新用子类引用指向子类对象：向下转型

  • 为了使用子类中的方法

多态的缺点

当子类对象向上转为父类类型后，编译器不能调用子类中特有的方法；

解决办法：需要调用子类的方法时，可以把父类类型强制转化为子类类型

==instanceof==

final关键字

可以用于声明 属性、方法和类

  • final修饰的类不能被其他类继承！
  • final 修饰的成员方法不能被子类重写
  • final 修饰方法参数，则不能在方法中修改其值
  • final修饰成员变量，值不能改变

接口

interface关键字

接口的实现

  • 接口可以继承其他接口
  • 一个类可以实现多个接口
  • 接口与实现的子类之间也存在多态

动态绑定机制

编译类型和运行类型的概念

  • 当调用对象==方法==的时候，该方法会和该对象的==内存地址==（运行类型）绑定。
  • 当调用对象==属性==时，==没有动态绑定机制==，哪里声明，哪里使用。

内部类 inner class

一个类的内部嵌套包含了另一个类的结构

分类
  • 定义在外部类的局部位置上
    1. 局部内部类（有类名）
    2. 匿名内部类（无类名）
  • 定义在外部类的成员位置上
    1. 成员内部类（无static修饰）
    2. 静态内部类（static修饰）
局部内部类
  1. 局部内部类定义在外部类的局部位置，通常在方法里
  2. 内部类可以直接访问外部类的所有成员，包含私有的
  3. 不能添加访问修饰符，但是可以使用==final==修饰。因为局部内部类本质也是局部变量，局部变量都可以用final
  4. 作用域：尽在定义它的方法或代码块中
  5. 局部内部类访问外部类的成员：直接访问
  6. 外部类访问内部类成员：创建内部类对象，再访问（==必须在作用域内==）
  7. 如果外部类和局部内部类的成员重名时，默认遵循就近原则(内部的)，如果想访问外部类的成员，则可以使用 ==外部类名.this.成员== 去访问
匿名内部类
  1. 本质仍然是一个类 内部类 该类没有名字 同时还是一个对象
  2. 定义在外部类的局部位置

  • 底层会分配类名

    为： ==外部类名$1==

    如： Tiger$1

    数字会随匿名类的个数改变

  • jdk底层在创建匿名内部类时，立刻创建了一个实例，并把地址返回

  • 匿名内部类使用一次就不能再使用，==注意区别这个对象还可以使用==
